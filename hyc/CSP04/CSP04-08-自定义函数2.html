思考问题：下面代码输出什么？ 
void addOne(int x) { x++; }
int a = 5;
addOne(a);
cout << a;   // 输出？
答案：5，不是6！
因为是值传递，函数操作的是 a 的副本！
如果想让函数真正修改外部变量，需要用引用传递或指针传递！

1️⃣ 函数默认参数

函数参数可以有默认值，调用时可以省略：

// 定义：有默认值的参数必须放在最后面
int add(int a, int b = 10, int c = 0) {
    return a + b + c;
}

// 调用：
add(5);          // a=5, b=10, c=0   → 15
add(5, 3);       // a=5, b=3,  c=0   → 8
add(5, 3, 2);    // a=5, b=3,  c=2   → 10
⚠️ 规则：默认参数必须从右向左连续设置！不能跳过——如果 b 有默认值但 c 没有，则不合法。

默认参数：常见用法

输出分隔符（默认空格，可指定换行）：

void printArr(int* a, int n, char sep = ' ') {
    for (int i = 0; i < n; i++) {
        cout << a[i];
        if (i < n - 1) cout << sep;
    }
    cout << endl;
}

printArr(a, n);        // 空格分隔（默认）
printArr(a, n, '\n');  // 换行分隔
💡 好处：一个函数实现多种输出格式，代码复用性更高！

2️⃣ 三种参数传递方式对比

// 值传递：修改的是副本
void doubleVal(int x) { x = x * 2; }

// 引用传递：直接修改原变量
void doubleRef(int& x) { x = x * 2; }

// 指针传递：通过地址修改原变量
void doublePtr(int* x) { *x = (*x) * 2; }
方式 语法 是否复制 能否修改原变量
值传递 f(int x) ✅ 复制 ❌ 不能
引用传递 f(int& x) ❌ 不复制 ✅ 能
指针传递 f(int* x) ❌ 不复制 ✅ 能（通过 *）
const 引用 f(const int& x) ❌ 不复制 ❌ 不能

3️⃣ 经典应用：swap 函数

❌ 值传递版（错误！）

void swapWrong(int a, int b) {
    int t = a; 
    a = b; 
    b = t;   // 只交换了副本！
}

✅ 引用传递版（正确）

void mySwap(int& a, int& b) {
    int t = a; 
    a = b; 
    b = t;   // 交换原变量！
}
mySwap(x, y);  // x 和 y 被真正交换
指针传递版（C 风格）： 
void mySwapPtr(int* a, int* b) {
    int t = *a; *a = *b; *b = t;
}
mySwapPtr(&x, &y);  // 传地址

4️⃣ 函数在竞赛题目中的应用

封装常用操作（代码复用）

// 封装：判断一个数是否是素数
bool isPrime(int n) {
    if (n < 2) return false;
    for (int i = 2; i * i <= n; i++)
        if (n % i == 0) return false;
    return true;
}

// 在主程序中多次调用
for (int i = 2; i <= 100; i++)
    if (isPrime(i)) cout << i << " ";
💡 好处：把复杂逻辑封装起来，主程序简洁清晰，易于调试和维护！

⚡ GESP 真题演练

抢答题 1

函数定义 void f(int a, int b=5, int c) 的问题是（ ）

A. 函数名不合法
B. 默认参数必须从右到左连续，c 没有默认值不能放在 b 后面
C. 参数个数太多
D. 返回值类型错误
答案：B
解析：默认参数必须从右到左连续——如果 b 有默认值，那 b 右边的所有参数（c）也必须有默认值，否则调用时二义性。

⚡ GESP 真题演练

判断题 2

引用传递和值传递的区别在于：引用传递函数内的修改会影响原变量，值传递不会。（ ）

答案：✓（正确）
解析：引用 int& x 是原变量的别名，任何对 x 的修改等同于修改原变量；值传递 int x 是副本，修改副本不影响原变量。
抢答题 3

以下哪种参数类型最适合"只需读取一个很大的结构体，不需要修改它"的场景？

A. Student s（值传递）
B. Student& s（引用传递）
C. const Student& s（const 引用）
D. Student* s（指针传递）
答案：C
解析：A 有拷贝开销；B 可修改原变量（不符合"只读"）；C 无拷贝开销且只读，最合适；D 也无拷贝但需要传地址，语义不如 C 直观。

📚 进阶扩展：函数重载

同名函数因参数类型或数量不同而共存：

int add(int a, int b) { return a + b; }
double add(double a, double b) { return a + b; }
int add(int a, int b, int c) { return a + b + c; }

add(1, 2);        // 调用 int 版本
add(1.5, 2.5);    // 调用 double 版本
add(1, 2, 3);     // 调用三参数版本
静态多态：编译器根据参数的类型和数量自动选择合适的重载版本。

练习 3：用引用传参返回多个值
综合

编写函数 void findMinMax(int* a, int n, int& minVal, int& maxVal) 同时返回数组的最小值和最大值。

void findMinMax(int* a, int n, int& minVal, int& maxVal) {
    minVal = maxVal = a[0];
    for (int i = 1; i < n; i++) {
        if (a[i] < minVal) minVal = a[i];
        if (a[i] > maxVal) maxVal = a[i];
    }
}

🌟 课堂总结

1️⃣ 默认参数

  • 有默认值的参数必须放在右边
  • 调用时从右往左可以省略
  • 必须从右向左连续设置

2️⃣ 三种传参方式

  • 只读小数据 → 值传递
  • 需要修改 → 引用传递
  • 只读大数据 → const 引用
  • 传数组 → 指针传递
掌握这些，你就能写出更高质量的函数代码！
